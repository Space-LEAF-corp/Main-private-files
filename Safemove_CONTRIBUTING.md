# 🌱 Contributing to Space LEAF Corp Projects

Thank you for your interest in contributing to Space LEAF Corp.  
We welcome collaborators who share our commitment to **digital safety**, **ethical technology**, and **community‑centered innovation**.

This document explains how to contribute effectively and responsibly.

---

## 🧭 Our Principles

All contributions must align with Space LEAF Corp’s core values:

- **Safety First** — Protecting users, especially minors, from harm.
- **Ethical Technology** — No exploitation, manipulation, or harmful design.
- **Transparency** — Clear communication, open processes, and honest intent.
- **Inclusivity** — Respect for all contributors and users.
- **Quality** — Clean, maintainable, well‑documented work.

If a contribution conflicts with these principles, it may be declined or revised.

---

## 🛠 Ways to Contribute

### **1. Report Issues**
Use GitHub Issues to report:
- Bugs  
- Incorrect detection behavior  
- Documentation improvements  
- Feature requests  
- Safety concerns  

Please include:
- A clear description  
- Steps to reproduce (if applicable)  
- Expected vs. actual behavior  

### **2. Submit Pull Requests**
Pull Requests (PRs) are welcome for:
- Code improvements  
- New detection patterns  
- Documentation updates  
- Educational content additions  
- API enhancements  

Before submitting a PR:
- Ensure your code is clean and readable  
- Add comments where needed  
- Include tests if applicable  
- Follow the style guidelines below  

### **3. Improve Documentation**
We welcome contributions to:
- README.md  
- CONTACT.md  
- Safety guides  
- Curriculum materials  
- API documentation  

Clear documentation helps everyone.

### **4. Suggest New Safety Features**
If you have ideas for:
- New red‑flag categories  
- Additional educational modules  
- Platform‑level safety policies  
- Detection improvements  

Open an Issue or start a Discussion.

---

## 🧪 Development Setup

1. Fork the repository  
2. Clone your fork  
3. Install dependencies  
4. Run tests (if included)  
5. Make your changes  
6. Submit a Pull Request
