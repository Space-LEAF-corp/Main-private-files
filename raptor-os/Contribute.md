🛠️ RAPTOR OS CONTRIBUTOR GUIDE

How to Contribute to a Guardian‑Grade OS

---

1. Before You Contribute

Read these documents:

• Raptor OS Manifesto
• Raptor OS Philosophy
• Developer Code of Honor
• Ceremonial OS Design Principles


If these don’t resonate with you, Raptor OS is not your ecosystem.

---

2. Contribution Types

You can contribute:

• code
• documentation
• creature personalities
• movement prompts
• STEM lessons
• museum modules
• classroom templates
• artifact stories
• accessibility improvements


All contributions must pass emotional‑safety review.

---

3. How to Submit

3.1 Fork the repo

Keep your branch names calm and descriptive.

3.2 Follow the Style Guide

UI, creature behavior, tone, pacing.

3.3 Run Safety Checks

Use the Safety API for all user‑generated content.

3.4 Submit a Pull Request

Include:

• purpose
• mode(s) affected
• safety considerations
• testing notes
• ceremonial impact (if any)


3.5 Pass Review

Two approvals required:

• Technical Steward
• Emotional‑Safety Steward


---

4. What Not to Submit

• competitive mechanics
• monetization systems
• ads
• tracking
• fast animations
• fear‑based content
• predatory behavior
• dark patterns
• addictive loops


These violate the Code of Honor.

---

5. Contributor Oath

I contribute with care, clarity, and stewardship.
I protect imagination.
I honor emotional safety.
I design for calm.
I build for children.
I leave the trail better than I found it.

---
