# MicroTech Chat: Dictation Stabilizer
# Code of Ethics (v2.0)

This module exists to correct dictation drift without erasing identity,
dialect, culture, or tone. All contributors must follow these ethical
requirements.

---

## 1. Identity Preservation

The system must never:
- Change pronouns without explicit user correction
- Alter names or identity markers
- Remove or sanitize cultural slang
- Force “standard English” unless the user requests it

Identity is not an error.

---

## 2. Cultural Respect

Dialects are valid linguistic systems.
Contributors must not treat:
- AAVE
- Southern English
- Caribbean English
- Spanish-influenced English
- Creole blends
- Any regional or cultural speech

as “incorrect” or “broken.”

Refinement must preserve cultural voice.

---

## 3. Transparency

All corrections must be:
- Visible
- Explainable
- Reversible

No silent changes. No hidden normalization.

---

## 4. Local-First Privacy

User data must:
- Stay on-device by default
- Never be used for training without explicit consent
- Be deletable at any time

---

## 5. User-Controlled Learning

The system learns only when:
- The user accepts corrections
- The user enables learning mode

No automatic profiling.

---

## 6. Tone and Intent Protection

The engine must preserve:
- Humor
- Emotion
- Rhythm
- Slang
- Personal phrasing

Clarity must not come at the cost of personality.

---

## 7. Ethical Development Practices

Contributors must:
- Avoid prescriptive grammar bias
- Respect linguistic diversity
- Provide dialect-inclusive test cases
- Document all model changes
- Prioritize user intent over algorithmic assumptions

If you cannot uphold these values, do not contribute.

---

## 8. Purpose

This module exists to support people whose voices are often misunderstood by
technology. Our responsibility is to build tools that honor every voice,
not force conformity.

This is the standard for all work in this repository.
