# Contributing to MicroTech Chat: Dictation Stabilizer

Thank you for your interest in contributing to the Dictation Stabilizer project.
This module supports dialect speakers, multilingual speakers, and users whose
natural voice is often misinterpreted by mainstream dictation systems.

Contributions are welcome, but all collaborators must follow the guidelines
below to ensure the system remains ethical, identity-safe, and culturally
respectful.

---

## How to Contribute

### 1. Fork and Branch
- Fork the repository.
- Create a feature branch using a descriptive name:
  - `feature/dialect-recognition-update`
  - `fix/pronoun-lock-bug`
  - `docs/add-spanish-tests`

### 2. Follow the Project Principles
All contributions must align with:
- Identity preservation
- Cultural respect
- Transparency in corrections
- Local-first processing
- User-controlled learning

See `CODE_OF_ETHICS.md` for full details.

### 3. Add Tests
Every change must include tests for:
- Dialect stability
- Pronoun/name safety
- Multilingual handling
- Tone preservation

### 4. Document Your Work
If your change affects behavior, update:
- `README.md`
- `VERSION.md`
- Relevant architecture or API documentation

### 5. Submit a Pull Request
Include:
- A clear description of the change
- Why it is needed
- Test results
- Any known limitations

---

## Code Style

- Keep functions small and readable.
- Use descriptive variable names.
- Avoid hidden transformations.
- Never introduce silent normalization of dialect or slang.

---

## Communication

Be respectful, clear, and collaborative.
This project serves real people with real voices—treat the work with care.

Thank you for helping build a system that understands every voice.
