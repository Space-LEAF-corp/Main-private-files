# Contributing to GQI&EA

Thank you for your interest in contributing to the GQI&EA system!
This project values clarity, emotional safety, and respectful collaboration.
Please review the guidelines below before submitting changes.

---

## How to Contribute

### 1. Fork the Repository
Create your own fork of the project and work from a feature branch.

### 2. Create a Feature Branch
Use descriptive names such as:
- `feature/tag-generator-improvements`
- `fix/timestamp-format`
- `docs/public-description-update`

### 3. Follow the Coding Style
- Keep functions small and readable.
- Use clear naming conventions.
- Add comments where logic may not be obvious.
- Avoid introducing unnecessary dependencies.

### 4. Add Tests When Possible
If you modify core logic, include or update tests in the `tests/` directory.

### 5. Document Your Changes
If your contribution affects public behavior or user-facing features:
- Update relevant files in the `docs/` folder.
- Add examples if needed.

### 6. Submit a Pull Request
Include:
- A clear description of what you changed
- Why the change is needed
- Any relevant screenshots or examples

---

## Code of Conduct

By participating in this project, you agree to:
- Communicate respectfully
- Provide constructive feedback
- Avoid harassment or discrimination
- Support an emotionally safe environment for all contributors

Violations may result in removal from participation.
